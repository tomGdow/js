Introduction

  1. The String object is used to represent and manipulate a sequence of characters
  2. Some of the most-used operations on strings are
    1. to check their length 
      "mystr".length // 5
    2. to build and concatenate them using the + and += string operators 
      let str = "joe"
str += "doe"
console.log(str) //joedoe
    3. check for the existence or location of substrings with the indexOf() method 
      "mystr".indexOf("s") // 2
    4. extract substrings with the substring() method 
      "mystr".substr(1,2) // "ys"

String Generation

  1. Strings can be created as primitives from string literals 
    const s1 = "A str primitive"
const s2 = 'A string primitive'
const s3 = `A string primitive`
  2. as objects using the String() constructor 
    const s4 = new String("A str obj")
  3. with String("mystr") without the new keyword (which produces a string primitive)

Character access

  1. "cat".charAt(1) // "a"
  2. "cat"[1] // "a"

String primitives and String objects

  1. String literals (denoted by double or single quotes) and strings returned from String calls in a non-constructor context (that is, called without using the new keyword) are primitive strings
  2. A String object can always be converted to its primitive counterpart with the valueOf() method 
    mystr= new String("2+2")  
    // [String: '2+2']

typeof mystr  // 'object'

eval(mystr)+100 // '2+2100'

eval(mystr.valueOf())+100
   // 104

String Coercion

  1. Strings are returned as-is.
  2. undefined turns into "undefined"
  3. null turns into "null"
  4. true turns into "true";
  5. false turns into "false"
  6. Numbers are converted with the same algorithm as toString(10)
  7. Symbols throw a TypeError
  8. Objects are first converted to a primitive by calling (in the following order)
    1. [Symbol.toPrimitive]() (with "string" as hint)
    2. toString()
    3. valueOf()
    The resulting primitive is then converted to a string

String Coercion Methods

  1. Template literal: `${x}` does string coercion
  2. String()
  3. Using the + operator
    1. "" + x coerces its operand to a primitive instead of a string, and, for some objects, has entirely different behaviors from normal string coercion
