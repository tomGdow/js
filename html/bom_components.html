window

  1. The window object serves a dual purpose in the browsers
    1. The javaScript interface to the browser window
    2. The ECMAScript global object
  2. The open() method of the window interface loads a specified resource into a new or existing browsing context (that is, a tab, a window, or an iframe) under a specified name
    1. open()
    2. open(url)
    3. open(url, target)
    4. open(url, target, windowFeatures)
    5. url
      1. A string indicating the URL or path of the resource to be loaded
      2. If an empty string ("") is specified or this parameter is omitted, a blank page is opened into the targeted browsing context
      target
      1. A string, without whitespace, specifying the name of the browsing context the resource is being loaded into
      2. If the name doesn't identify an existing context, a new context is created and given the specified name
      3. The following special target keywords can also be used
        1. _self
        2. _blank (default)
        3. _parent
        4. _top
        5. _unfencedTop
      features
      A string containing a comma-separated list of window features in the form name=value
      Boolean values can be set to true using one of
      1. name
      2. name=yes
      3. name=true
      4. name=n (where n is any non-zero integer)
      These features include options such as the window's default size and position, whether or not to open a minimal popup window, and so forth
    6. Caution
      It is preferable to avoid resorting to window.open(), for several reasons
      1. Modern browsers offer a popup-blocking feature
      2. Modern browsers offer tab-browsing, and tab-capable browser users prefer opening new tabs to opening new windows
      3. Never use window.open()inline in HTML

Navigator

  1. Originally introduced in Netscape Navigator 2, the navigator object is the standard for browser identification on the client
  2. The navigator object is common among all JavaScript-enabled web browsers
    1. .appName
    2. .appVersion
    3. .buildId
    4. .cookieEnabled
    5. .oscpu
    6. .platform
    7. .userAgent

Location

  1. The location object provides information about the document that is currently loaded in the window, as well as general navigation functionality
  2. The location object is unique in that it is a property of both window and document
  3. Both window.location and document.location point to the same object
  4. The location object also parses the URL into discrete segments that can be accessed via location propterties
    1. .hash
    2. .host
    3. .hostname
    4. .href
    5. .pathname
    6. .port
    7. .protocol
    8. .search
    9. .username
    10. .password
    11. .origin

Screen

  1. The screen object (also a property of window) is one of the few JavaScript objects that have little to no programmatic use
  2. It is used purely as an indication of client capabilities
  3. This object provides information about the client's display outside the browser window, including information such as pixel width and height
    Selected Functions
    1. .availHeight
    2. .availWidth
    3. .height
    4. .width
    5. .orientation

History

  1. The history object represents the user's navigation history since the given window was first used
  2. Because history is a property of window, each browser window object has its own history object
  3. For security reasons, it is not possible to determine the URLs that the user has visited
  4. It is possible, however, to navigate backwards and forwards through the list of places the user has been without knowing the exact URL
  5. history.go(-1)
