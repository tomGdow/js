Keywords

  1. Keywords are tokens that look like identifiers but have special meanings in JavaScript
  2. For example, the keyword async before a function declaration indicates that the function is asynchronous
  3. Some keywords are reserved, meaning that they cannot be used as an identifier for variable declarations, function declarations, etc
  4. They are often called reserved words
  5. Not all keywords are reserved — for example, async can be used as an identifier anywhere
  6. Some keywords are only contextually reserved — for example,
    1. await is only reserved within the body of an async function
    2. let is only reserved
      1. in strict mode code
      2. in const and let declarations
  7. Identifiers are always compared by string value, so escape sequences are interpreted
