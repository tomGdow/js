Introduction

  1. The const declaration declares block-scoped local variables
  2. The value of a constant can't be changed through reassignment using the assignment operator, but if a constant is an object, its properties can be added, updated, or removed

Notes

  1. The const declaration is very similar to let
  2. const declarations are scoped to blocks as well as functions
  3. const declarations can only be accessed after the place of declaration is reached (temporal dead zone)
    1. For this reason, const declarations are commonly regarded as non-hoisted
  4. const declarations do not create properties on globalThis when declared at the top level of a script
  5. const declarations cannot be redeclared by any other declaration in the same scope
  6. const begins declarations, not statements
    1. That means you cannot use a lone const declaration as the body of a block
  7. An initializer for a constant is required
    1. You must specify its value in the same declaration
  8. The const declaration creates an immutable reference to a value
    1. It does not mean the value it holds is immutable â€” just that the variable identifier cannot be reassigned
    2. For instance, in the case where the content is an object, this means the object's contents (e.g., its properties) can be altered
  9. You should understand const declarations as
    1. "create a variable whose identity remains constant", not "whose value remains constant"
    2. or, "create immutable bindings", not "immutable values"
