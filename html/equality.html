Introduction

  1. The equality (==) operator checks whether its two operands are equal, returning a Boolean result
  2. Unlike the strict equality operator, it attempts to convert and compare operands that are of different types
  3. The most notable difference between this operator and the strict equality (===) operator is that the strict equality operator does not attempt type conversion
  4. Instead, the strict equality operator always considers operands of different types to be different

Comparison Description

  1. If the operands have the same type
    1. Object: return true only if both operands reference the same object
    2. String: return true only if both operands have the same characters in the same order
    3. Number: return true only if both operands have the same value
      +0 and -0 are treated as the same value
    4. If either operand is NaN, return false
      so, NaN is never equal to NaN
    5. Boolean: return true only if operands are both true or both false.
  2. If one of the operands is null or undefined, the other must also be null or undefined to return true
    Otherwise return false
  3. If one of the operands is an object and the other is a primitive, convert the object to a primitive
  4. At this step, both operands are converted to primitives (one of String, Number, Boolean, Symbol)
    1. If one of the operands is a Boolean but the other is not, convert the boolean to a number
    2. Number to String: convert the string to a number
      Conversion failure results in NaN, which will guarantee the equality to be false

why is str1 == str2?
  1. str → a primitive string value: "hello"
  2. str2 → a String object: String9"hello")
  3. This is not a primitive; its an object with properties and methods
  4. == (loose equality) in JavaScript performs type coercion if the operands are of different types
  5. When comparing a primitive to an object, the object is converted to a primitive using ToPrimitive (which, for a String object, calls .valueOf( and returns "hello").
  6. After coercion: "hello" == "hello" → true li> Final result: true
  7. But === (strict equality) would give false because it does not coerce types
