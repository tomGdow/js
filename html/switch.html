Introduction

  1. The switch statement evaluates an expression, matching the expression's value against a series of case clauses, and executes statements after the first case clause with a matching value, until a break statement is encountered
  2. The default clause of a switch statement will be jumped to if no case matches the expression's value
  3. Switch uses the strict equality (===) for comparison
  4. fall-through: If break is omitted, execution will proceed to the next case clause, even to the default clause, regardless of whether the value of that clause's expression matches

Description

  1. A switch statement first evaluates its expression
  2. It then looks for the first case clause whose expression evaluates to the same value as the result of the input expression, using the strict equality comparison and transfers control to that clause, executing all statements following that clause
  3. The clause expressions are only evaluated when necessary — if a match is already found, subsequent case clause expressions will not be evaluated, even when they will be visited by fall-through
  4. If no matching case clause is found, the program looks for the optional default clause, and if found, transfers control to that clause, executing statements following that clause.
  5. If no default clause is found, the program continues execution at the statement following the end of switch.
  6. By convention, the default clause is the last clause, but it does not need to be so
  7. A switch statement may only have one default clause
  8. Multiple default clauses will result in a SyntaxError

Syntax


switch (expression) {

  case caseExpression1:
    statements

  case caseExpression2:
   statements

  // …

  case caseExpressionN:
    statements

  default:
   statements
}

Code Example

[JavaScript Inserts] 
const foo = "Apples"

switch(foo.toLowerCase()) {

  case "oranges":
    console.log("orange")
    break;

  case "apples":
    console.log("Apples");
  break;

default:
  console.log("None of above");

}
