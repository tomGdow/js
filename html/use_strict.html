Introduction

  1. JavaScript's strict mode is a way to opt in to a restricted variant of JavaScript, thereby implicitly opting-out of "sloppy mode"
  2. Strict mode isn't just a subset: it intentionally has different semantics from normal code
  3. Strict mode code and non-strict mode code can coexist, so scripts can opt into strict mode incrementally

Benefits of Strict Mode

  1. Eliminates some JavaScript silent errors by changing them to throw errors
  2. Fixes mistakes that make it difficult for JavaScript engines to perform optimizations: strict mode code can sometimes be made to run faster than identical code that's not strict mode
  3. Prohibits some syntax likely to be defined in future versions of ECMAScript

Invoking strict mode

  1. Strict mode applies to entire scripts or to individual functions
  2. It doesn't apply to block statements enclosed in {} braces: attempting to apply it to such contexts does nothing
  3. To invoke strict mode for an entire script, put the exact statement "use strict"; (or 'use strict';) before any other statements
  4. Likewise, to invoke strict mode for a function, put the exact statement "use strict"; (or 'use strict';) in the function's body before any other statements
  5. The entire contents of JavaScript modules are automatically in strict mode, with no statement needed to initiate it
  6. All parts of a class's body are strict mode code, including both class declarations and class expressions

Comment

  1. "use strict"; makes this default to undefined in functions (not the global object)
  2. "use strict"; makes assignment to an undeclared variable a ReferenceError
