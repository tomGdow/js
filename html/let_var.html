Introduction

  1. The let declaration declares re-assignable, block-scoped local variables, optionally initializing each to a value

Comparison with var

  1. let declarations are scoped to blocks as well as functions
  2. let declarations can only be accessed after the place of declaration is reached (see temporal dead zone)
  3. For this reason, let declarations are commonly regarded as non-hoisted
  4. let declarations do not create properties on globalThis when declared at the top level of a script
  5. let declarations cannot be redeclared by any other declaration in the same scope
  6. let begins declarations, not statements
    1. That means you cannot use a lone let declaration as the body of a block
  7. Note that let is allowed as an identifier name when declared with var or function in non-strict mode, but you should avoid using let as an identifier name to prevent unexpected syntax ambiguities
  8. Many style guides (including MDN's) recommend using const over let whenever a variable is not reassigned in its scope
    1. This makes the intent clear that a variable's type (or value, in the case of a primitive) can never change
    2. Others may prefer let for non-primitives that are mutated
  9. The list that follows the let keyword is called a binding list and is separated by commas, where
    1. the commas are not comma operators
    2. the = signs are not assignment operators
    3. initializers of later variables can refer to earlier variables in the list

Scope

The scope of a variable declared with let is one of the following curly-brace-enclosed syntaxes that most closely contains the let declaration

  1. block statement
  2. switch statement
  3. try...catch statement
  4. body of one of the for statements, if the let is in the header of the statement
  5. function body
  6. static initialization block
  7. Or if none of the above applies
    1. the current module, for code running in module mode
    2. the global scope, for code running in script mode

Notes

  1. A let declaration within a function's body cannot have the same name as a parameter
  2. A let declaration within a catch block cannot have the same name as the catch-bound identifier
