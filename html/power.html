Introduction

  1. The exponentiation (**) operator returns the result of raising the first operand to the power of the second operand
  2. It is equivalent to Math.pow(), except it also accepts BigInts as operands
  3. The exponentiation operator is right-associative:
    a ** b ** c is equal to a ** (b ** c)
  4. The ** coerces both operands to numeric values and tests the types of them
  5. NaN ** 0 (and the equivalent Math.pow(NaN, 0)) is the only case where NaN doesn't propagate through mathematical operations
  6. NaN ** 0 returns 1 despite the operand being NaN
  7. 0 raised to a positive power returns 0
  8. 0 raised to a power of 0 returns 1
  9. 0 raised to a negative number returns Infinity
  10. -0 raised to a negative number returns -Infinity
  11. In JavaScript, it is impossible to write an ambiguous exponentiation expression
  12. That is, you cannot put a unary operator (with precedence 14, including +/-/~/!/++/--/delete/void/typeof/await) immediately before the base number; doing so will cause a SyntaxError
  13. For example, -2 ** 2 is 4 in Bash, but is -4 in other languages (such as Python)
  14. This is invalid in JavaScript, as the operation is ambiguous
  15. You have to parenthesize either side — for example, as -(2 ** 2) — to make the intention unambiguous
  16. In effect this means that
    1. an unparenthesized unary expression can't appear on the left-hand side of '**'
    2. but is OK on the right-hand side of **
    3. 2 ** -2 is OK
    4. but -2 ** 2 produces a SyntaxError
