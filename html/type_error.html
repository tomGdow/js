Introduction

  1. TypeError is a runtime error in JavaScript
  2. It occurs when an operation is performed on an existing value of an inappropriate type
  3. In essence, TypeError is a runtime error that happens when you try to do something with a value that's not of the right type for that operation — even though the syntax is valid
  4. Common causes include trying to access properties of null or undefined, or calling a non-function as a function
  5. It's JavaScript's way of saying: "I understand what you're asking, but this kind of value can't do that"
  6. TypeErrors can be caught and handled using try-catch blocks

Key Points

  1. TypeError not a syntax error — the code is written correctly but fails while running.
  2. Its about mismatched value types, not about incorrect spelling or missing variables (those are usually ReferenceErrors).
  3. The message after TypeError: usually tells you the exact problem.

Code Example

[JavaScript Inserts] 
try {
  
  // Code that may throw a TypeError

  null.foo();

} catch (error) {

  if (error instanceof TypeError) {

    console.log("Caught a TypeError:", 
    error.message);

  } else {

    throw error; // rethrow if it's not a TypeError

  }
}
