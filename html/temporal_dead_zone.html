Introduction

  1. A variable declared with let, const, or class is said to be in a "temporal dead zone" (TDZ) from
    1. the start of the block
    2. until code execution reaches the place where the variable is declared and initialized
  2. While inside the TDZ, the variable has not been initialized with a value, and any attempt to access it will result in a ReferenceError
  3. The variable is initialized with a value when execution reaches the place in the code where it was declared
  4. If no initial value was specified with the variable declaration, it will be initialized with a value of undefined
  5. This differs from var variables, which will return a value of undefined if they are accessed before they are declared
  6. The term "temporal" is used because the zone depends on the order of execution (time) rather than the order in which the code is written (position)

    // TDZ starts at beginning of scope

    var bar = 1;
