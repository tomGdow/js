Introduction

  1. The Date() constructor creates Date objects
  2. When called as a function, it returns a string representing the current time

new Date(value)
new Date(dateString)
new Date(dateObject)

Date()

Parameters

There are five basic forms for the Date() constructor
  1. No parameters
    1. When no parameters are provided, the newly-created Date object represents the current date and time as of the time of instantiation
    2. The returned date's timestamp is the same as the number returned by Date.now()
  2. value
    1. An integer value representing the timestamp
    2. The timestamp is the number of milliseconds since midnight at the beginning of January 1, 1970, UTC
    3. Also known as the epoch
  3. datestring
    1. A string value representing a date, parsed and interpreted using the same algorithm implemented by Date.parse()
  4. dateobject
    1. An existing Date object
    2. This effectively makes a copy of the existing Date object with the same date and time
    3. This is equivalent to new Date(dateObject.valueOf()), except the valueOf() method is not called
  5. Individual date and time component values
    1. Given at least a year and month, this form of Date()returns a Date object whose component values 
      (year, 
 month, 
 day, 
 hour, 
 minute, 
 second,
 millisecond)
      all come from the following parameters
    2. Any missing fields are given the lowest possible value (1 for day and 0 for every other component)
    3. The parameter values are all evaluated against the local time zone, rather than UTC
    4. Date.UTC() accepts similar parameters but interprets the components as UTC and returns a timestamp (NOT a date object!)
    5. If any parameter overflows its defined bounds, it "carries over"
    6. For example, if a monthIndex greater than 11 is passed in, those months will cause the year to increment
    7. Similarly, if any parameter underflows, it "borrows" from the higher positions.
    8. For example, new Date(2020, 5, 0) will return May 31st, 2020
    9. year
      1. Integer value representing the year
      2. Values from 0 to 99 map to the years 1900 to 1999
      3. All other values are the actual year
    10. monthIndex
      1. Integer value representing the month
      2. Beginning with 0 for January to 11 for December
    11. day
      1. Integer value representing the day of the month
      2. Defaults to 1
      3. Optional
    12. hours
      1. Integer value between 0 and 23 representing the hour of the day
      2. Defaults to 0
      3. Optional
    13. minutes
      1. Integer value representing the minute segment of a time
      2. Defaults to 0
      3. Optional
    14. seconds
      1. Integer value representing the second segment of a time
      2. Defaults to 0
      3. Optional
    15. milliseconds
      1. Integer value representing the minute segment of a time
      2. Defaults to 0
      3. Optional
  6. Date()
    1. Calling the Date() function without the new keyword returns a string representation of the current date and time, exactly as new Date().toString() does
    2. Any arguments given in a Date() function call (without the new keyword) are ignored; regardless of whether it's called with an invalid date string â€” or even called with any arbitrary object or other primitive as an argument
    3. Calling Date() without the new keyword always returns a string representation of the current date and time
