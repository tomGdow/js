Introduction

  1. In JavaScript, function scope means that variables declared inside a function are only accessible within that function (and its nested functions), not outside it
  2. Anything declared with var, let, or const inside a function is scoped to that function
  3. Outside code cannot directly access those variables
  4. Inner (nested) functions can still access them — this is the basis of closures
  5. var is function-scoped, meaning it ignores block boundaries (if, for, etc.) but still respects the containing function
  6. let and const are block-scoped, but a block inside a function is still part of that function's scope
  7. Function scope is different from global scope (variables declared outside all functions) and block scope

Summary

Function scope
Variables declared with var, let, or const inside a function are hidden from the outside world
Blockscope
let and const live only inside the { ... } block they are declared in; var ignores this and stays in the function's scope
Closures
Inner functions can reach up to their parent's variable

Function Arguments

  1. Arguments exist only inside the function body
  2. They are not accessible outside that function
  3. Inside the function, you can access them by their name or via the special arguments object (in non-arrow functions)
  4. If a variable with the same name is declared in an inner scope (or in parameters themselves), it shadows outer variables.
  5. Default values for parameters are evaluated in their own scope (the parameter scope), which is separate from the function body scope
  6. You can reference earlier parameters in later parameter defaults, but not variables declared inside the function body.
  7. In non-strict mode, parameters and the arguments object are linked: changing one changes the other.
  8. In strict mode, they are decoupled
  9. Arrow functions don't have their own arguments object — they inherit it from their enclosing scope

Shadowing

  1. In JavaScript, shadowing happens when a variable declared within a certain scope (e.g., inside a function or block) has the same name as a variable in an outer scope
  2. The inner declaration “shadows” the outer one — meaning inside that scope, the inner variable takes precedence, and the outer variable becomes inaccessible by its name 
let name = "Outer";

function example() {

  let name = "Inner"; 
      // This shadows the outer 'name'

  console.log(name); // "Inner"
}

example();

console.log(name); // "Outer"
  1. Shadowing can happen with function scope (var) or block scope (let / const)
  2. It can also occur with parameters shadowing outer variables 
let value = 10;

function test(value) { // parameter shadows 
                          the outer variable

  console.log(value); // uses the parameter
}

test(5); // 5
  1. You can shadow let with another let (or const) in a nested block
  2. You cannot shadow a let or const with var in the same or overlapping scope
  3. You can shadow var with let or const inside a block (because block-scoped variables don't leak out)

Illegal Shadowing

  1. Illegal shadowing in JavaScript happens when you try to redeclare a variable in the same scope using var when the original was declared with let or const
  2. JavaScript treats this as a syntax error because let and const are block-scoped and cannot be overridden by a var (which is function-scoped and hoisted)
  3. Note that the following IS allowed! 

    var x = 10;
var x = 100;
